

1 - Instalar o Java
    site: https://www.oracle.com/java/technologies/downloads/#jdk22-mac
    arquivo: https://download.oracle.com/java/22/latest/jdk-22_macos-x64_bin.dmg (sha256)
    Verificar se está instalado: java --version

2 - Instalar o Node
    site: https://nodejs.org/en
    Verificar se está instalado: node --version

3 - Instalando o Appium
    Entre no site do Codeceptjs https://codecept.io > Guides > Mobile Testing
    3.1 - instale o appium doctor com o comando: sudo npm i -g appium-doctor (utilize o sudo para desbloquear com senha)
    3.2 - rode o appium doctor no terminal com o comando: appium-doctor (para verificar o sistema)
    3.3 - instle o appium com o comando: sudo npm i -g appium
    3.4 - instale o Appium desktop
        3.4.1 repositório: https://github.com/appium/appium-desktop/releases/tag/v1.22.3-4
        3.4.2 arquivo: https://github.com/appium/appium-desktop/releases/download/v1.22.3-4/Appium-Server-GUI-mac-1.22.3-4.dmg
            OBS: depois de tentar abrir uma vez o appium desktop será necessário entrar em 
            'preferências do sistema' > 'segurança e privacidade' (e autorizar o sistema a abrir o Appium desktop)

4 - Instalar o Xcode
    Para o Macbook 2017 a versão do xcode disponível é a Xcode 14.2, para baixar-la é necessário:
    4.1 - entrar no site: https://developer.apple.com/br/support/xcode/
    4.2 - acessar a página: https://developer.apple.com/download/all/?q=xcode
        obs: será necessário entrar com a conta (e-mail) e a senha do iCloud (senha geral)
    4.3 - pesquise a baixe o Xcode 14.2

5 - Instalar Android Studio
    5.1 - baixar o Android Studio no site: https://developer.android.com/studio?hl=pt-br

6 - Configurar as variáveis de ambiente do Java e Android no mac
    5.1 - encontre o caminho do java com o comando: /usr/libexec/java_home
    5.2 - abra o arquivo 'nano' para configurar as variáveis com o comando: nano ~/.zshrc  (obs: estou utilizando o terminal zshrc)
    5.3 - adicione as variáveis de ambiente:
        export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.x.x_xx.jdk/Contents/Home
        export PATH=$JAVA_HOME/bin:$PATH
        5.3.1 - obs: /Library/Java/JavaVirtualMachines/jdk1.x.x_xx.jdk/Contents/Home --> substitua pelo caminho encontrado no item 5.1
        
        5.3.2 - Exemplo de como ficou as variáveis de ambiente do java e Android (exemplo real)

                export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-22.jdk/Contents/Home
                export PATH=$JAVA_HOME/bin:$PATH

                export ANDROID_HOME=/Users/rodrigorossetitozatti/Library/Android/sdk
                export PATH=$PATH=$ANDROID_HOME/tools:$ANDROID_HOME/emulator
                export PATH=$PATH=$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools
                export PATH=$PATH=$ANDROID_HOME/tools:$ANDROID_HOME/tools

    5.4 - saia do nano com os comandos: 
        control + x 
        y
        Enter 
    5.5 - salve as confogurações utilizando o comando: source ~/.zshrc
    5.6 - para garantir que a configuração está correta execute o comando: echo $JAVA_HOME
        o resultado tem que ser o caminho do java
