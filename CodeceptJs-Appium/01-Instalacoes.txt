
1 - Git: https://git-scm.com/
    Depois de instalado configurar nome e e-mail no prompt de comando
        git config --global user.name "seu nome" (obs: seu nome entre aspas)
        git config --global user.email "seu e-mail" (obs: seu e-mail entre aspas)

            Para testar: git config user.name
                         git config user.email

    site: https://nodejs.org/en
    Verificar se está instalado: node --version

5 - Instalando o Appium
    Entre no site do Codeceptjs https://codecept.io > Guides > Mobile Testing
    5.1 - instale o appium doctor com o comando: sudo npm i -g appium-doctor (utilize o sudo para desbloquear com senha)
    5.2 - rode o appium doctor no terminal com o comando: appium-doctor (para verificar o sistema)
    5.3 - instle o appium com o comando: sudo npm i -g appium
    5.4 - instale o Appium desktop
        5.4.1 repositório: https://github.com/appium/appium-desktop/releases/tag/v1.22.3-4
        5.4.2 arquivo: https://github.com/appium/appium-desktop/releases/download/v1.22.3-4/Appium-Server-GUI-mac-1.22.3-4.dmg
            OBS: depois de tentar abrir uma vez o appium desktop será necessário entrar em 
            'preferências do sistema' > 'segurança e privacidade' (e autorizar o sistema a abrir o Appium desktop)

6 - Instalar o Xcode
    Para o Macbook 2017 a versão do xcode disponível é a Xcode 14.2, para baixar-la é necessário:
    6.1 - entrar no site: https://developer.apple.com/br/support/xcode/
    6.2 - acessar a página: https://developer.apple.com/download/all/?q=xcode
        obs: será necessário entrar com a conta (e-mail) e a senha do iCloud (senha geral)
    6.3 - pesquise a baixe o Xcode 14.2

7 - Instalar Android Studio
    7.1 - baixar o Android Studio no site: https://developer.android.com/studio?hl=pt-br

8 - Configurar as variáveis de ambiente do Java e Android no mac
    8.1 - encontre o caminho do java com o comando: /usr/libexec/java_home
    8.2 - abra o arquivo 'nano' para configurar as variáveis com o comando: nano ~/.zshrc  (obs: estou utilizando o terminal zshrc)
    8.3 - adicione as variáveis de ambiente:
        export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk1.x.x_xx.jdk/Contents/Home
        export PATH=$JAVA_HOME/bin:$PATH
        8.3.1 - obs: /Library/Java/JavaVirtualMachines/jdk1.x.x_xx.jdk/Contents/Home --> substitua pelo caminho encontrado no item 5.1
        
        8.3.2 - Exemplo de como ficou as variáveis de ambiente do java e Android (exemplo real)

                export JAVA_HOME=/Library/Java/JavaVirtualMachines/jdk-22.jdk/Contents/Home
                export PATH=$JAVA_HOME/bin:$PATH

                export ANDROID_HOME=/Users/rodrigorossetitozatti/Library/Android/sdk
                export PATH=$PATH=$ANDROID_HOME/tools:$ANDROID_HOME/emulator
                export PATH=$PATH=$ANDROID_HOME/tools:$ANDROID_HOME/platform-tools
                export PATH=$PATH=$ANDROID_HOME/tools:$ANDROID_HOME/tools

    8.4 - saia do nano com os comandos: 
        control + x 
        y
        Enter 
    8.5 - salve as confogurações utilizando o comando: source ~/.zshrc
    8.6 - para garantir que a configuração está correta execute o comando: echo $JAVA_HOME
        o resultado tem que ser o caminho do java
